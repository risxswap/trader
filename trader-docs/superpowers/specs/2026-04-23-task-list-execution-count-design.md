# 任务列表执行次数统计设计

## 1. 背景

当前任务管理页已经能够展示任务基础信息、运行状态、执行结果、参数和更新时间，但还缺少一个用于快速判断任务活跃度的核心指标：

- 当前看不到某个任务累计执行过多少次

如果需要了解任务历史活跃度，只能进入任务详情页或任务日志页间接判断，路径较长，也不利于在任务列表中做批量对比。因此本次需求是在任务列表页中直接增加“执行次数”统计。

## 2. 目标

- 在任务管理页增加单独一列 `执行次数`
- 统计口径为该任务历史执行记录的累计总数
- 前端列表加载后即可直接展示，无需逐条额外查询
- 不影响现有任务列表分页、筛选、编辑和执行功能

## 3. 统计口径

### 3.1 统计规则

执行次数定义为：

- 该任务在任务日志表中的历史记录总数
- 不区分 `RUNNING`、`SUCCESS`、`FAILED`
- 只要存在一条执行历史，就计为 1 次

### 3.2 展示规则

- 有历史记录时显示对应数字
- 无历史记录时显示 `0`
- 不附加“次”字样，保持列表紧凑和便于扫表

## 4. 匹配设计

### 4.1 系统任务

系统任务列表中的任务实例已具备：

- `taskCode`
- `taskName`
- `taskType`

而当前任务日志查询逻辑中，日志记录与任务实例的主要映射关系已经按：

- 任务实例 `taskCode`
- 任务日志 `taskGroup`

进行关联。

因此本次执行次数统计沿用相同口径：

- 按任务实例 `taskCode`
- 聚合任务日志表中 `task_group = taskCode` 的记录总数

这样能够与现有任务详情页按任务维度查看历史的行为保持一致。

### 4.2 策略任务

任务管理页支持混合展示策略任务，但策略任务当前使用 `sourceType=INVESTMENT` 的拼装模型，和任务日志表不一定存在稳定的一一映射关系。

为避免误统计：

- `sourceType=INVESTMENT` 的记录默认返回 `executionCount = 0`
- 本次不额外扩展策略任务日志统计能力

## 5. 后端设计

### 5.1 DTO 扩展

在 `SystemTaskDto` 中新增字段：

- `executionCount`

用于承载任务列表中的累计执行次数。

### 5.2 服务层聚合

任务列表接口保持现有入口不变：

- `POST /task/list`

在 `SystemTaskService.list()` 中，在取得当前页系统任务记录后，执行一次批量聚合统计：

- 收集当前页系统任务的 `taskCode`
- 查询任务日志表中这些 `taskCode` 对应的执行记录数量
- 将统计结果回填到 `SystemTaskDto.executionCount`

### 5.3 DAO 层能力

需要在任务日志 DAO 中增加按任务编码批量统计执行次数的能力，例如：

- 输入：`taskCode` 列表
- 输出：`taskCode -> count`

该聚合应在数据库层一次完成，避免在服务层对每个任务单独查询。

### 5.4 性能约束

统计范围只针对当前页系统任务，不做全量任务统计。

这样可以：

- 控制 SQL 聚合范围
- 避免任务列表分页越大时请求数量线性膨胀
- 保持列表加载性能在可控范围内

## 6. 前端设计

### 6.1 列表展示

在任务管理页表格中新增一列：

- `执行次数`

位置建议放在：

- `执行结果` 之后
- `任务参数` 之前

这样更符合“状态 -> 结果 -> 历史规模 -> 参数”的阅读顺序。

### 6.2 展示样式

- 单元格居中展示
- 值为纯数字
- 默认显示 `0`

不额外引入 Tag、Tooltip 或颜色状态，保持表格信息密度稳定。

## 7. 错误处理

- 统计查询失败时，应整体按任务列表接口失败处理，避免只返回半成品数据
- 若某个任务没有聚合结果，则回填 `0`
- 策略任务无匹配关系时直接显示 `0`，不视为异常

## 8. 测试策略

### 8.1 后端测试

至少覆盖：

- `SystemTaskService` 返回任务列表时包含 `executionCount`
- 多个任务时能够正确按 `taskCode` 分别回填统计值
- 没有日志记录的任务返回 `0`
- 策略任务返回 `0`

### 8.2 前端验证

至少覆盖：

- 任务管理页新增 `执行次数` 列
- 列表中能展示接口返回的统计值
- 没有值时显示 `0`
- 新增列不影响现有操作列布局

## 9. 验收标准

- 任务管理页新增单独一列 `执行次数`
- 执行次数按任务历史记录累计总数统计
- 当前页系统任务的执行次数能正确显示
- 没有执行历史的任务显示 `0`
- 策略任务默认显示 `0`
- 不破坏现有任务列表筛选、分页、编辑、执行和详情能力
