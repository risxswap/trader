# 模块：节点管理

## 目标与范围

管理系统中的各类节点设备，主要包括执行节点（Executor Node）和数据采集节点（Collector Node）。这些节点会定时向系统的心跳中心（Redis）上报其在线状态以及设备硬件统计信息。为了方便历史状态查询、性能分析和审计，这些上报数据也会同步持久化到 ClickHouse 中。

本模块提供节点列表展示、节点状态监控、硬件资源统计查询等功能。

## 业务需求与规则

### 节点类型
- **执行节点 (EXECUTOR)**：负责执行投资策略和定时调度任务。
- **数据采集节点 (COLLECTOR)**：负责从外部接口（如行情数据源）抓取基础数据。

### 数据流转
1. **实时状态 (Redis)**
   - 节点每隔固定时间（如 10 秒）向 Redis 发送心跳。
   - Redis 中维护当前所有存活节点的最新状态及最近一次的硬件指标，并设置 TTL 以实现节点掉线自动剔除（或标记为离线）。
2. **历史数据 (ClickHouse)**
   - 后端服务（或节点自身）将心跳与硬件统计数据批量异步写入 ClickHouse。
   - ClickHouse 存储包含时间戳的明细数据，用于后续的节点稳定性分析、硬件瓶颈排查。

### 上报信息 (节点指标)
每次心跳上报的数据需包含：
- **基本信息**：节点ID (node_id)、节点名称 (node_name)、节点类型 (node_type: EXECUTOR / COLLECTOR)、IP 地址、应用版本号。
- **硬件统计**：
  - CPU 使用率 (cpu_usage, e.g., 45.5%)
  - 内存使用率 (memory_usage, e.g., 60.2%)
  - 磁盘使用率 (disk_usage, e.g., 80.1%)
  - 网络流量（可选，上行/下行速率）
- **时间信息**：心跳时间戳 (timestamp)。

## 前端设计（admin-web）

### 页面入口

- **节点列表与状态大屏**：`/node/list`
  - 页面组件：`src/pages/node/List.vue`

### 关键交互

1. **节点概览展示**
   - 顶部提供数据统计卡片：当前在线节点数、离线节点数、CPU/内存负载过高的告警节点数。
2. **节点列表视图**
   - 展现方式：支持卡片视图与表格视图切换。
   - 列表字段：节点ID、节点名称、类型、状态（在线/离线，通过最后心跳时间判断）、IP地址、当前CPU、当前内存、版本号、最后心跳时间。
   - 过滤条件：按节点类型、在线状态、名称搜索。
3. **节点详情与历史图表**
   - 点击节点进入详情弹窗或页面。
   - 通过 ECharts 展示该节点近 24 小时或 7 天的 CPU、内存使用率历史走势（数据源为 ClickHouse）。

## 后端设计（admin-server）

### 接口定义

- Controller：`NodeController.java`
- Service：`NodeService.java`

**主要接口：**
1. **GET `/api/nodes`**
   - 作用：获取当前所有节点的实时状态列表。
   - 数据源：从 Redis 读取。
   - 响应结构：返回 `List<NodeStatusDto>`，包含节点的基本信息和最新的硬件使用率。
2. **GET `/api/nodes/{nodeId}/history`**
   - 作用：查询指定节点在某个时间段内的硬件历史指标。
   - 请求参数：开始时间、结束时间。
   - 数据源：从 ClickHouse 查询聚合或明细数据。
   - 响应结构：返回时间序列数组，用于前端 ECharts 渲染。

### 数据存储设计

#### Redis (实时状态)
- **Key 结构**：`node:status:{nodeId}`
- **Value 类型**：Hash 或 JSON String，存储 `NodeStatusDto` 的完整信息。
- **TTL**：如设置为 30 秒（心跳间隔的 3 倍），超时即认为离线；或者使用 Sorted Set 记录最近心跳时间。

#### ClickHouse (历史指标表 `node_metrics_log`)
- `timestamp` (DateTime) - 心跳时间，可作为排序键。
- `node_id` (String) - 节点唯一标识。
- `node_type` (String) - EXECUTOR / COLLECTOR。
- `cpu_usage` (Float32) - CPU使用率。
- `memory_usage` (Float32) - 内存使用率。
- `disk_usage` (Float32) - 磁盘使用率。
- **索引与引擎**：使用 `MergeTree` 引擎，按 `node_id` 和 `timestamp` 建立主键或排序键。
