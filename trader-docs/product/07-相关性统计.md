# 模块：相关性统计

## 目标与范围

相关性统计模块用于计算并展示基金/标的之间的相关性结果，支持：

- 查询相关性结果（分页）
- 手动触发两只标的的相关性计算
- 定时全量计算（基金两两组合，固定周期 1Y）

## 前端设计（admin-web）

### 页面入口

- 路由：/analysis/correlation
- 页面：[CorrelationList.vue](../../trader-admin/admin-web/src/pages/analysis/CorrelationList.vue)

### 查询交互

- 查询条件：asset1、asset2、period、minCoefficient、maxCoefficient
- 数据来源：GET /correlation（分页）
- 表格字段：asset1/asset2、类型、coefficient、period、createdAt/updatedAt

### 接口定义

- 查询相关性
  - Path：GET /correlation
  - Request：CorrelationQuery（pageNo/pageSize + asset1/asset2/period + coefficient 区间）
  - Response：ResData<PageDto<CorrelationDto>>
  - 处理逻辑：CorrelationDao.pageQuery 按 created_at 倒序分页

- 手动计算
  - Path：POST /correlation/calculate
  - Request：CorrelationCalculateParam（asset1、asset2、period）
  - Response：ResData<Void>
  - 处理逻辑：CorrelationService.calculateAndSave(asset1, asset2, period)

- CRUD（补充能力）
  - POST /correlation：新增（CorrelationParam）
  - PUT /correlation：更新（CorrelationParam，要求 id 非空）
  - DELETE /correlation/{id}：删除

### 相关性计算逻辑

CorrelationService.calculateAndSave 的核心步骤：

1. 计算起始时间 startTime
   - period 支持：数字天数、xD/xM/xY；空值默认 1Y
2. 读取两只标的的 FundNav（从 startTime 起，按 time 升序）
   - 数据字段：FundNav.adjNav（复权净值）
3. 以 date 对齐两条序列，仅保留共同日期
4. Pearson 相关系数 r：
   - 使用 Apache Commons Math：PearsonsCorrelation
5. 显著性检验 p 值：
   - t = r * sqrt((n-2)/(1-r^2))
   - 2*(1 - TDistribution(n-2).cdf(|t|))
6. 过滤阈值：
   - |r| > 0.5 且 p < 0.05 才入库
7. 按 (asset1, asset2, period) 唯一键 upsert：
   - 已存在：更新 coefficient/pValue/updatedAt
   - 不存在：插入并补齐 asset1Type/asset2Type（从 fund.type 读取）

## 定时任务与并发模型

- 定时任务：每天 02:00 触发全量计算
  - cron：0 0 2 * * ?
  - 处理策略：
    - 读取已上市基金列表（FundDao.listAll）
    - 组合两两不重复配对（i<j + canonical key 去重）
    - 将每个配对的计算任务提交到 correlationExecutor
- 并发执行器：correlationExecutor（线程池）
  - corePoolSize=5, maxPoolSize=10, queueCapacity=500

## 数据模型（MySQL）

表定义参考：[db.sql](../../trader-admin/admin-server/src/main/resources/database/db.sql)

### correlation

- id（PK）
- asset1、asset1_type
- asset2、asset2_type
- coefficient（相关系数）
- p_value（显著性 p 值）
- period
- created_at、updated_at

### fund_nav（计算输入）

- symbol + time（联合主键）
- adj_nav（复权净值）

## 约束与扩展点

- 当前计算使用“复权净值水平值”直接计算相关性；若要更符合金融时间序列统计，通常改为“收益率序列”（例如对数收益率）再计算相关性。
- 定时全量计算的组合数量为 O(N^2)，基金数量增长后需要：
  - 分片/分批（按 symbol 前缀、区间等）
  - 降低频率或只对活跃标的计算
  - 将结果缓存并增量更新（仅新基金/新净值区间触发）
