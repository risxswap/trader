# 模块：消息推送

## 目标与范围

消息推送模块提供统一的消息发送能力，并记录发送日志。当前实现支持企业微信机器人（WeCom Webhook），主要用于投资日志（investment_log）通知。

## 前端设计（admin-web）

### 页面设计

前端需实现推送日志的查询列表与详情展示：

- 路由：/msg-push-log
- 页面：`Index.vue`
- API：[log.ts](../../trader-admin/admin-web/src/services/log.ts)
  - `GET /msg-push-log`（分页查询日志）

### 页面交互

- **筛选区**：支持按 `type` (例如 investment_log)、`channel`、`status` (SUCCESS/FAIL) 进行下拉筛选，支持基于日期的筛选。
- **列表区**：表格展示推送记录，包含：
  - 标题（Title）
  - 类型（Type）
  - 渠道（Channel）
  - 状态（Status，以 Tag 标签展示绿红）
  - 创建时间（Created At）
  - 操作列（查看原文）
- **详情弹窗**：点击“查看原文”打开 Dialog，使用只读的 Markdown 渲染组件或直接显示 `content` 字段内容；展示 `recipient` 信息及返回的响应结果（若失败则展示报错原因）。

### 接口定义

- 推送日志查询
  - Path：GET /msg-push-log
  - Request：MsgPushLogQuery（pageNo/pageSize + type/channel/status）
  - Response：ResData<PageDto<MsgPushLogDto>>
  - 处理逻辑：按 created_at 倒序分页返回

- 手动发送消息
  - Path：POST /msg-push-log/send
  - Request：MsgPushParam（type/title/content/channel）
  - Response：ResData<Void>
  - 处理逻辑：当前忽略 channel，统一调用 WeComChannel.pushText

### 企业微信通道设计

WeComChannel 的处理流程：

1. 从配置读取 webhook-url（fund.wecom.webhook-url）
2. 组装企业微信机器人 payload
   - 文本：{ msgtype: "text", text: { content } }
   - Markdown：{ msgtype: "markdown", markdown: { content } }
3. HTTP POST 发送到 webhook-url
4. 解析返回 JSON 的 errcode 判定 SUCCESS/FAIL
5. 持久化写入 msg_push_log（无论成功失败）

配置位置：
- application.yml：fund.wecom.webhook-url（可通过环境变量 WECOM_WEBHOOK_URL 覆盖）
  - [application.yml](../../trader-admin/admin-server/src/main/resources/application.yml)

### 投资日志通知（事件驱动）

- 订阅频道：investment_log
  - 频道枚举：[Channels.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/pubsub/Channels.java)
- 触发流程：
  1. 外部系统向 Redis channel=investment_log 发布 InvestmentMessage（JSON，包含 id=investmentLogId）
  2. InvestmentLogSubscriber 收到消息后调用 InvestmentLogService.notifyInvestmentLog(investmentLogId)
  3. InvestmentLogService 生成 Markdown（资产/现金/盈亏 + 交易记录 + 持仓记录）
  4. WeComChannel.pushMarkdown 发送企业微信消息，并落库 msg_push_log
  5. InvestmentLog.notified 置为 1，避免重复推送

## 数据模型（MySQL）

表定义参考：[db.sql](../../trader-admin/admin-server/src/main/resources/database/db.sql)

### msg_push_log

- id（PK）
- type（消息类型，例如 investment_log）
- content（消息体）
- status（SUCCESS/FAIL）
- channel（当前为 WeCom）
- title、recipient
- created_at、updated_at

## 约束与扩展点

- MsgPushLogService.send 当前仅支持企业微信文本；如要支持多渠道（Email/SMS/钉钉等），建议抽象 Channel 接口并按 param.channel 路由。
- WeComChannel 默认会记录完整 content 到 msg_push_log；如消息内容可能包含敏感信息，需要对入库内容做脱敏或增加开关控制。
