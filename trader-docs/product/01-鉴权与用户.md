# 模块：鉴权与用户

## 目标与范围

本模块提供管理端登录与用户密码修改能力，供 admin-web 建立会话状态与进行基础的账号维护。

## 前端设计（admin-web）

### 页面入口

- 登录页：/login
  - 页面实现：[Login.vue](../../trader-admin/admin-web/src/pages/Login.vue)

### 关键交互与状态

- 登录表单：username、password，Element Plus 表单校验通过后发起登录请求
- 登录成功后持久化：
  - localStorage.token：后端返回的 token 字符串
  - localStorage.user：后端返回的用户对象（username/nickname）
- 登录成功跳转：/dashboard
- 路由守卫：未登录访问受限页面时重定向到 /login
  - 路由实现：[index.ts](../../trader-admin/admin-web/src/router/index.ts)

### API 调用

- POST /auth/login（登录）
- POST /user/change-password（修改密码；从主布局用户下拉触发）
  - 布局实现：[MainLayout.vue](../../trader-admin/admin-web/src/layouts/MainLayout.vue)

## 后端设计（admin-server）

### 接口与职责分层

- Controller：
  - 登录：[AuthController.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/controller/AuthController.java)
  - 修改密码：[UserController.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/controller/UserController.java)
- Service：
  - 登录校验与 token 生成：[AuthService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/AuthService.java)
  - 初始化管理员与修改密码：[UserService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/UserService.java)
- Dao：
  - 用户查询与更新：UserDao（路径见工程结构）

### 接口定义

- 登录
  - Path：POST /auth/login
  - Request：LoginParam（username、password）
  - Response：ResData<LoginDto>
  - 处理逻辑：
    - 按 username 查询 users
    - locked=true 直接失败
    - BCrypt.checkpw 校验密码 hash
    - 更新 last_login
    - 返回随机 UUID token（当前用于前端会话标识）

- 修改密码
  - Path：POST /user/change-password
  - Request：ChangePasswordParam（username、oldPassword、newPassword）
  - Response：ResData<Void>
  - 处理逻辑：
    - 按 username 查询 users
    - locked=true 或 oldPassword 校验失败返回失败
    - BCrypt.hashpw 生成新 hash 并更新 password、updated_at

### 初始化逻辑

- 系统启动后确保存在管理员账号（仅在不存在时创建）
  - 启动器：[SystemInitializer.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/system/SystemInitializer.java)
  - 创建逻辑：[UserService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/UserService.java)

## 数据模型

### 表：users

- 表定义：[db.sql](../../trader-admin/admin-server/src/main/resources/database/db.sql)
- 关键字段：
  - id：主键
  - username：登录名（唯一性需由数据库或业务约束保证）
  - password：BCrypt hash
  - nickname：展示昵称
  - locked：是否锁定
  - last_login：最近登录时间
  - created_at / updated_at：审计字段

## 统一响应与异常

- 统一响应体：ResData（code、message、data、timestamp）
  - 实现：[ResData.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/common/model/ResData.java)
- 错误码枚举：ErrorCode
  - 实现：[ErrorCode.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/common/model/ErrorCode.java)
- 全局异常处理：Warning / 校验异常 / IllegalArgumentException
  - 实现：[ExceptionAdvice.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/advice/ExceptionAdvice.java)

## 关键流程（时序）

### 登录

1. 前端提交 username/password 到 POST /auth/login
2. AuthService 校验用户存在、未锁定、BCrypt 密码匹配
3. 更新 users.last_login
4. 返回 LoginDto（含 token）
5. 前端写入 localStorage.token 与 localStorage.user，跳转 /dashboard

### 修改密码

1. 前端提交 username/oldPassword/newPassword 到 POST /user/change-password
2. UserService 校验用户存在、未锁定、旧密码匹配
3. 生成新 hash 更新 users.password 与 updated_at
4. 返回 success

## 已知约束与扩展点

- 当前 token 为随机 UUID，服务端未实现基于 token 的鉴权拦截与校验；前端路由守卫仅依赖 localStorage.token。
- 如需真正的鉴权链路，建议补充：登录态存储（Redis/DB）、请求拦截器（Header 携带 token）、权限模型（角色/菜单/接口权限）。
