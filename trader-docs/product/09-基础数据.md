# 模块：基础数据

## 目标与范围

基础数据模块为前端提供通用下拉选项与标的查询能力，降低各业务模块对底层数据源细节的耦合。

当前包含：

- 策略列表（用于投资模块的策略选择与动态参数表单）
- 标的列表（用于搜索标的代码；当前仅支持基金/ETF）

## 前端设计（admin-web）

### 使用场景

- 投资计划新增/编辑时：
  - 调用 GET /basic/strategies 获取策略列表
  - 调用 GET /basic/symbols 获取标的选项（带 keyword 搜索）

相关页面：
- [investment/Detail.vue](../../trader-admin/admin-web/src/pages/investment/Detail.vue)

## 后端设计（admin-server）

### 分层与关键类

- Controller：[BasicController.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/controller/BasicController.java)
- Cache：[StrategyInfoCache.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/cache/StrategyInfoCache.java)
- Service（symbols 数据源）：[FundService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/FundService.java)

### 接口定义

- 策略列表
  - Path：GET /basic/strategies
  - Response：ResData<List<StrategyInfoDto>>
  - 处理逻辑：从 Redis Hash（key=STRATEGY_INFO）取 values 并反序列化为 StrategyInfoDto
  - DTO：
    - className：策略类名/唯一标识
    - name：策略展示名
    - configSchame：策略参数 schema（字符串形式，通常为 JSON Schema）
    - config：策略参数默认值/当前值（字符串）
    - [StrategyInfoDto.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/common/model/dto/StrategyInfoDto.java)

- 标的查询
  - Path：GET /basic/symbols
  - Query：
    - type：标的类型（例如 FUND/ETF）
    - keyword：可选，模糊过滤 symbol/name
  - Response：ResData<List<Map<String,String>>>
    - value：symbol
    - label："symbol - name"
  - 处理逻辑：
    - type=FUND 或 ETF：FundService.listAll() 全量取出后在内存过滤与 limit(50)
    - 其他 type：返回空列表

## 缓存模型（Redis）

### STRATEGY_INFO（Hash）

- key：STRATEGY_INFO
- field：StrategyInfoDto.className
- value：StrategyInfoDto 的 JSON 字符串

写入端：

- StrategyInfoCache.cacheStrategies(List<StrategyInfoDto>)：批量写入 Hash
- StrategyInfoCache.getStrategy(className)：按 field 读取
- StrategyInfoCache.getAllStrategies()：读取所有 values 并反序列化

实现：
- [StrategyInfoCache.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/cache/StrategyInfoCache.java)

## 约束与扩展点

- symbols 查询当前采用“全量拉取后内存过滤”，数据量增大时建议下沉到数据库侧条件查询（新增 FundDao.searchSymbols 等）或增加缓存。
- 策略信息的写入来源不在本工程内实现；需要外部系统/初始化流程将策略元数据写入 STRATEGY_INFO，否则 /basic/strategies 将返回空列表。
