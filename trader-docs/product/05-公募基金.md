# 模块：公募基金

## 目标与范围

公募基金模块用于维护基金基础信息、提供基金列表与详情查询，并通过净值数据展示净值走势。

## 前端设计（admin-web）

### 页面入口

- 公募基金列表：/funds/list
  - 页面：[List.vue](../../trader-admin/admin-web/src/pages/funds/List.vue)
- 公募基金详情：/funds/detail/:symbol
  - 页面：[Detail.vue](../../trader-admin/admin-web/src/pages/funds/Detail.vue)
- 基金净值：/funds/nav
  - 页面：[NavList.vue](../../trader-admin/admin-web/src/pages/funds/NavList.vue)

### 列表页交互

- 查询条件：keyword（代码/名称）、management、custodian、fundType、管理费/托管费区间
- 固定条件：market='O'
- 排序：
  - managementFee/custodianFee/listDate：服务端排序（sortBy/sortOrder）
  - updatedAt：前端对当前页数据排序
- 行为：
  - 详情：跳转 /funds/detail/:symbol
  - 编辑：弹窗编辑 name/status/market/exchange，提交后刷新列表
  - 删除：二次确认后删除并刷新列表

### 详情页交互

- 远程搜索基金代码：调用列表接口（pageSize=10）用于下拉选项
- 过滤条件：symbol + 日期范围
- 净值走势图：
  - 拉取净值数据后按日期升序排序
  - 使用 ECharts 折线图展示 unitNav/accumNav/adjNav
  - dataZoom 触发“向左/向右加载更多”逻辑（每次 90 天，合并去重）

### API 调用

- POST /fund/list（基金分页列表）
- GET /fund/detail/{symbol}（基金详情）
- PUT /fund/update/{symbol}（更新基金基础信息）
- DELETE /fund/delete/{symbol}（删除基金）
- POST /fund-nav/list（净值分页列表，用于详情页图表与净值列表）
  - API 实现：[fund.ts](../../trader-admin/admin-web/src/services/fund.ts)

## 后端设计（admin-server）

### 接口与职责分层

- Controller：
  - 基金：[FundController.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/controller/FundController.java)
  - 净值：[FundNavController.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/controller/FundNavController.java)
- Service：
  - 基金：[FundService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/FundService.java)
  - 净值：[FundNavService.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/service/FundNavService.java)
- Dao：
  - FundDao / FundNavDao / FundAdjDao / FundMarketDao（MyBatis-Plus）

### 接口定义

- 基金分页列表
  - Path：POST /fund/list
  - Request：FundListQuery（pageNo/pageSize/keyword/market/management/custodian/fundType/fee 区间/sortBy/sortOrder）
  - Response：ResData<PageDto<FundDto>>
  - 处理逻辑：FundDao.pageQuery -> FundService.listFunds 映射为 FundDto

- 基金详情
  - Path：GET /fund/detail/{symbol}
  - Response：ResData<Fund>
  - 处理逻辑：FundDao.getByCode

- 更新基金
  - Path：PUT /fund/update/{symbol}
  - Request：FundUpdateParam（name/status/market/exchange）
  - Response：ResData<Void>
  - 处理逻辑：FundDao.updateBasicBySymbol，失败返回 400

- 删除基金
  - Path：DELETE /fund/delete/{symbol}
  - Response：ResData<Void>
  - 处理逻辑：FundDao.deleteBySymbol，未删除返回 404

- 净值分页列表
  - Path：POST /fund-nav/list
  - Request：FundNavListQuery（pageNo/pageSize/symbol/startTime/endTime）
  - Response：ResData<PageDto<FundNavDto>>
  - 处理逻辑：
    - FundNavDao.pageQuery 获取净值记录
    - 若指定 symbol：读取 FundAdj（复权因子）并构建日期 -> 因子映射
    - 计算 accumNav = unitNav + accumDiv（若 accumDiv 存在）
    - 计算 adjNav = unitNav * factor（使用 floorEntry 获取不晚于该日的最新因子）

## 数据模型（MySQL）

表定义参考：[db.sql](../../trader-admin/admin-server/src/main/resources/database/db.sql)

### fund

- symbol（PK）
- name、status、market、exchange
- management、custodian、fund_type
- management_fee、custodian_fee
- list_date、found_date
- created_at、updated_at

### fund_nav

- symbol + time（联合主键）
- unit_nav、accum_nav、accum_div
- net_asset、total_net_asset
- created_at、updated_at

### fund_adj

- symbol + time（联合主键）
- adj_factor
- created_at、updated_at

## 关键流程（时序）

### 详情页净值走势渲染

1. 前端请求 GET /fund/detail/{symbol} 获取基金基本信息
2. 前端请求 POST /fund-nav/list 获取指定日期区间净值记录
3. 后端补齐 accumNav/adjNav 并返回
4. 前端按 time 升序排序并写入图表 series
5. dataZoom 接近边界时增量请求更多区间数据并合并去重

## 约束与扩展点

- listFunds 的排序能力依赖 FundDao.pageQuery 对 sortBy/sortOrder 的实现；前端对 updatedAt 的排序仅作用于当前页数据。
- FundNavService 对“早于首个复权因子日期”的记录默认不计算 adjNav（保持空值）；若业务需要，可定义更明确的缺省因子策略（例如 1.0 或首个因子）。
