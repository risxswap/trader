# 模块：投资

## 目标与范围

投资模块面向“投资计划”的全生命周期管理，包含券商、投资计划、资金日志、交易记录、持仓记录，并提供策略配置（动态表单）与日志推送能力。

### 核心交互设计

- 投资计划新增/编辑：
  - 券商下拉：调用 GET /broker（pageSize=200）
  - 策略下拉：调用 GET /basic/strategies
  - 策略参数动态表单：基于策略 schema.properties 渲染输入控件（字符串/数值等）
  - strategyConfig（后端字段）由前端将 strategyParams 序列化后提交（以页面实现为准）

- GET /investments：分页列表（InvestmentQuery）

- PUT /investments：更新（InvestmentParam）

业务规则（以 Service 实现为准）：
- brokerId 不能为空（新增时校验）
- status 缺省为 STOPPED
- cron / executorId 缺省为空字符串
- 新增/更新/删除会通过 Redis Pub/Sub 发布变更消息（investment 频道）

## 数据模型（MySQL）

表定义参考：[db.sql](../../trader-admin/admin-server/src/main/resources/database/db.sql)

### broker

- id（PK）
- name、code
- initial_capital、current_capital
- intro、remark
- created_at、updated_at

### investment

- id（PK）
- name、group_name
- target_type、invest_type
- broker_id（券商外键字段）
- targets（jsonb，标的列表）
- budget
- strategy、strategy_config
- cron、executor_id
- status（默认 STOPPED）
- created_at、updated_at

### investment_log

- id（PK）
- investment_id（索引 idx_investment_log_investment_id）
- type（默认 TRADE）
- record_date
- cash、asset、profit
- remark
- notified（0/1）
- created_at、updated_at

### investment_trading

- id（PK）
- investment_log_id（索引 idx_investment_trading_investment_log_id）
- investment_id（索引 idx_investment_trading_investment_id）
- asset、asset_type、volume、price
- created_at、updated_at

### investment_position

- id（PK）
- investment_id（索引 idx_investment_position_investment_id）
- investment_log_id（索引 idx_investment_position_investment_log_id）
- asset、asset_type
- quantity、buy_price、cost_price、side
- created_at、updated_at

## 关键流程（时序）

### 新增投资计划

1. 前端填写投资计划（券商、预算、策略、标的、策略参数）
2. POST /investments 提交 InvestmentParam
3. InvestmentService 校验 brokerId，补齐默认字段后写入 investment
4. 发布 Redis Pub/Sub 消息到 investment 频道（CREATE）

### 投资日志推送

1. 外部系统/任务向 Redis investment_log 频道发送包含 investmentLogId 的消息（InvestmentMessage）
2. InvestmentLogSubscriber 接收消息并调用 InvestmentLogService.notifyInvestmentLog
3. Service 读取 investment_log + 关联的 investment_trading / investment_position
4. WeComChannel 调用企业微信 webhook 推送，并写入 msg_push_log

## 校验与异常

- 参数校验：部分接口使用 @Valid（以 Controller 入参为准）
- 业务异常：InvestmentPositionService 中使用 Warning 抛出“记录不存在/删除失败”
  - [Warning.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/exception/Warning.java)
  - [ExceptionAdvice.java](../../trader-admin/admin-server/src/main/java/cc/riskswap/trader/admin/advice/ExceptionAdvice.java)

## 扩展点

- investment 频道当前仅发布未订阅：通常用于联动外部执行器（executor）按投资计划进行任务调度/更新缓存。
- 策略 schema 的来源依赖 StrategyInfoCache：需要有外部流程将策略信息写入 Redis（STRATEGY_INFO），前端才能生成动态参数表单。
